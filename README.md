# Контент-завод — The Global Wedding & Event

Автоматическая фабрика статей: по расписанию **сама пишет статьи, проверяет их и публикует** RSS-фид, который забирает Яндекс.Дзен. Бесплатно (Gemini + GitHub).

> ## ✅ Уже настроено под The Global Wedding
> - `config.json` — данные агентства (сайт, телефон 8-911-127-86-53, e-mail, ниша, аудитория).
> - Фирменный стиль обложек: чёрный фон + белый заголовок (шрифт Involve) + золотой акцент + логотип. Лого и шрифты лежат в `assets/`.
> - Промпты писателя/редактора адаптированы под свадьбы: тон от лица команды агентства, без слов «премиум/элитный», запрет выдуманных цен и названий площадок, регион — СПб и Лен. область.
> - `topics.json` — 25 тем под свадебные запросы (в т.ч. яхта, усадьба, лофт — закрывают пробелы SEO-ядра). Как пополнять — см. `sources.md`.
>
> ## ⬜ Осталось сделать тебе (10 минут, по шагам ниже)
> 1. Получить бесплатный ключ **Gemini** (шаг 1).
> 2. Создать **публичный репозиторий** на GitHub и залить туда эти файлы (шаг 4).
> 3. Вписать в `config.json` → `site_url` адрес своих GitHub Pages (шаг 4).
> 4. Положить ключ в **секреты** репозитория (шаг 5) и включить **Pages** (шаг 6).
> 5. Нажать **Run workflow** (шаг 8) — появится первая статья. Дальше — само.

## Что делает

```
По расписанию (напр. 3×/день) GitHub Actions:
  1. Берёт следующую тему из topics.json
  2. ПИСАТЕЛЬ (Gemini) пишет статью — живым языком, со ссылками на ваш сайт
  3. РЕДАКТОР (Gemini) проверяет: факты + анти-нейронный стиль
     FAIL → переписать по замечаниям → перепроверить
  4. PASS  → страница на GitHub Pages + брендовая обложка + запись в фид
     FAIL → откладывается в state.json (held) на ручную проверку
  5. Коммит результата обратно в репозиторий + деплой Pages
```

Итог: постоянный RSS-фид `https://ВАШ-АККАУНТ.github.io/ВАШ-РЕПО/feed.xml`, который подключается к Дзену.

## Файлы

| Файл | Что это |
|---|---|
| `config.json` | **все настройки проекта** (компания, продукты, ссылки, цвета) — правишь только его |
| `topics.json` | очередь тем статей |
| `article_factory.py` | движок: писатель + редактор-проверка + переписка |
| `run_factory.py` | один прогон: тема → статья → публикация |
| `covers.py` | рисует обложку (цвета/лого из config) |
| `gen_feed.py` | собирает RSS-фид (+ опц. блог Tilda) |
| `assets/` | сюда положить логотип-значок (см. файл внутри) |
| `.github/workflows/factory.yml` | расписание и автозапуск |
| `state.json`, `generated/` | автоматически: что уже опубликовано |

## Настройка (по шагам)

### 1. Ключ Gemini (бесплатно)
1. **aistudio.google.com** → войти Google-аккаунтом → **Get API key → Create API key**
2. Скопировать ключ (начинается с `AIza...`)

### 2. Заполнить `config.json`
Открой `config.json` и впиши свои данные: название компании, что продаёте (`niche_desc`), кто аудитория, ссылки (`home_url`, `catalog_url`), телефон, e-mail. Поле `site_url` заполнишь на шаге 4.
- `brand_display` — короткое название на обложке (напр. «БРЕНД»).
- `colors` — фирменные цвета фона/акцента (RGB).
- `tilda_rss` — если у вас блог на Tilda и хотите гнать и его статьи в Дзен: вставьте ссылку на RSS потока (иначе оставьте пустым — будут только авто-статьи).

### 3. Логотип
Положи в `assets/` файл `mark_white.png` (белый значок на прозрачном фоне). См. `assets/ПОЛОЖИ_СЮДА_ЛОГОТИП.txt`. Без него обложка тоже рисуется — только без значка.

### 4. Создать репозиторий на GitHub
1. Создай **публичный** репозиторий (GitHub Pages на бесплатном тарифе работает только из публичного; в репо нет секретов).
2. Впиши в `config.json` → `site_url`: `https://ТВОЙ-ЛОГИН.github.io/ИМЯ-РЕПО`
3. Залей все файлы в репозиторий (кроме `.env` — он в `.gitignore`).

### 5. Положить ключ в секреты
В репозитории: **Settings → Secrets and variables → Actions → New repository secret**
- Имя: `GEMINI_API_KEY`
- Значение: твой ключ `AIza...`

### 6. Включить Pages
**Settings → Pages → Build and deployment → Source: GitHub Actions.**

### 7. Расписание
В `.github/workflows/factory.yml` поле `cron` — время в **UTC**. По умолчанию `0 5,12,14 * * *` = 08/15/17 по Москве. Меняй под себя (crontab.guru).

### 8. Запуск и проверка
- **Actions → content-factory → Run workflow** (ручной прогон). Через ~2 мин появится статья.
- Открой `https://ТВОЙ-ЛОГИН.github.io/ИМЯ-РЕПО/feed.xml` — фид с первой статьёй.
- Дальше расписание работает само.

### 9. Подключить к Дзену (опционально)
Заявка на подключение RSS: **yandex.ru/support/zen/website/rss-connect.html** — указать канал + ссылку на `feed.xml`. После модерации статьи публикуются в Дзене сами.

## Локальный тест (до заливки)
```
pip install pillow
# создай файл .env со строкой:  GEMINI_API_KEY=твой_ключ
python article_factory.py "Тестовая тема статьи"   # проверить писателя+редактора
python run_factory.py                              # один полный прогон
python gen_feed.py public/feed.xml                 # собрать фид
```

## Особенности
- **Анти-нейронный стиль** зашит в промпты (без «рынок дышит», штампов, «правила трёх»).
- **Проверка фактов** — редактор ставит FAIL при выдуманных данных; такие статьи не публикуются, а ждут в `state.json` (held).
- Темы кончатся — добавь новые в `topics.json`.
- Публикация идёт на GitHub Pages → Дзен. На сайт (Tilda и т.п.) авто-постинг не делается (у большинства CMS нет API создания постов).
- Модель Gemini меняется в `config.json` (`gemini_model`).
